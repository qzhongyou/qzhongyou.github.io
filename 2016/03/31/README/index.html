git 操作指南

learn git

起步

  • 初次运行git配置

    • 用户信息
      • 每次 Git 提交时都会引用这两条信息，说明是谁提交了更新，所以会随更新内容一起被永久纳入历史记录。如果用了 –global选项，那么更改的配置文件就是位于你用户主目录下的那个，以后你所有的项目都会默认使用这里配置的用户信息。如果要在某个特定的项目中使用其他名字或者电邮，只要去掉 –global选项重新配置即可，新的设定保存在当前项目的 .git/config 文件里。
        $ git config --global user.name 'quezhongyou'和$ git config --global user.email '837300621@qq.com'
      • 查看配置信息，可以通过 git config --list
    • 生成shh密钥
      • 查看是否已经有了ssh密钥：cd ~/.ssh
      • 生成密钥$ ssh-keygen -t rsa -C “837300621@qq.com”
      • 在github上添加ssh密钥，这要添加的是“id_rsa.pub”里面的公钥
  • git status检查当前文件状态
    • 跟踪新文件
      • 添加新文件newfile,运行$ git status,出现Untracked files，表示为未跟踪文件。运行$ git add newfile.txt再运行$ git status,出现Changes to be committed，
        表示已暂存状态。
    • 跟踪暂存修改文件
      • 运行$ git status，出现Changes not staged for commit表示已修改内容，但是为添加到暂存区中。运行$ git add README.md再运行$ git status,出现Changes to be committed,
        表示已暂存状态。

  • .gitignore

  • git diff
    • $ git diff我们修改README.md文件，不运行$ git add README.md。查看尚未暂存的文件更新了哪些部分，直接输入 $ git diff.比较的是工作目录中当前文件和暂存区域快照之间的差异
      -我们修改README.md文件 +我们修改README.md文件,不运行。-为修改的内容，+为修改完成的内容。
    • git diff --cached已经暂存起来的文件和上次提交时的快照之间的差异（本地）
  • git commit
    • git commit -m提交。
    • git commit -a -m 跳过git add 这一步，直接commit
  • git rm
    • 从工作目录中手工删除文件。rm newfile.txt删除工作区文件，然后git rm newfile.txt删除暂存文件。git commit添加到本地仓库。
    • 如果删除之前修改过并且已经放到暂存区域的话，则必须要用强制删除选项 -f。
  • git mv
    • Git 中对文件改名。$ git mv file_from file_to,运行 git mv 就相当于运行了下面三条命令:
  • git log
    • git log查看提交历史。git log -p -2.-p 选项展开显示每次提交的内容差异，用 -2 则仅显示最近的两次更新。
    • --stat，仅显示简要的增改行数统计.
  • 取消已经暂存的文件和取消对文件修改
    • 进入暂存区的文件123.txt。执行git reset HEAD 123.txt,撤销暂存，再执行git status进行查看。
    • git checkout -- 123.txt,进行对文件的修改。这条命令有些危险，所有对文件的修改都没有了，因为我们刚刚把之前版本的文件复制过来重写了此文件。

  • git remote

    • git remote命令，它会列出每个远程库的简短名字。也可以加上 -v 选项（译注：此为 –verbose 的简写，取首字母），显示对应的克隆地址。
    • 添加远程仓库 $ git remote add node git@github.com:qzhongyou/node.git。
    • 重命名远程仓库。git remote rename dev redev。
    • 删除远程仓库。 git remote rm dev

    • git remote show [remote-name]查看某个远程仓库的详细信息.

      $ git config –global alias.br branch
      $ git config –global alias.ci commit
      $ git config –global alias.st status
      ```
      如果要输入 git commit 只需键入 git ci 即可
